# <a name="generate-er-reports-by-adding-content-as-raw-xml"></a><span data-ttu-id="f8e42-103">Gerar relatórios de ER adicionando conteúdo como XML bruto</span><span class="sxs-lookup"><span data-stu-id="f8e42-103">Generate ER reports by adding content as raw XML</span></span>

[!include[banner](../includes/banner.md)]

<span data-ttu-id="f8e42-104">Você pode usar o novo elemento de formato **XML BRUTO** para criar formatos os relatórios de eletrônicos (ER) que geram documentos de saída no formato XML.</span><span class="sxs-lookup"><span data-stu-id="f8e42-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="f8e42-105">Em alguns casos, talvez pode prefira adicionar dados XML brutos para esses relatórios por um ou mais destes motivos:</span><span class="sxs-lookup"><span data-stu-id="f8e42-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="f8e42-106">É mais conveniente usar o XML bruto para o projeto original e a manutenção em andamento de um relatório porque a estrutura XML pode ser gerada automaticamente executando uma expressão de tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="f8e42-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="f8e42-107">Portanto, várias associações não precisam ser determinadas para vários elementos do formato em tempo de design.</span><span class="sxs-lookup"><span data-stu-id="f8e42-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="f8e42-108">Isso é possível quando as fontes de dados que você estiver usando contiverem informações que podem ser usadas para criar elementos XML quando o relatório for gerado.</span><span class="sxs-lookup"><span data-stu-id="f8e42-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span> 
- <span data-ttu-id="f8e42-109">Nenhum outro método pode ser usado para preencher o relatório com o conteúdo XML que já foi recebido e armazenado no sistema.</span><span class="sxs-lookup"><span data-stu-id="f8e42-109">No other method can be used to fill the report with XML content that was previously received and stored in the system.</span></span> <span data-ttu-id="f8e42-110">Por exemplo, a resposta XML que é gerada talvez precise conter o conteúdo de uma solicitação XML que foi enviada anteriormente.</span><span class="sxs-lookup"><span data-stu-id="f8e42-110">For example, the XML response that is generated might have to contain the content of an XML request that was sent earlier.</span></span>
- <span data-ttu-id="f8e42-111">Nenhum outro método pode ser usado para inserir caracteres no documento gerado com base nos códigos numéricos.</span><span class="sxs-lookup"><span data-stu-id="f8e42-111">No other method can be used to insert characters into the generated document based on their numeric codes.</span></span> <span data-ttu-id="f8e42-112">Para alguns idiomas e caracteres, os códigos desse tipo não existem.</span><span class="sxs-lookup"><span data-stu-id="f8e42-112">For some languages and characters, codes of this type don't exist.</span></span> <span data-ttu-id="f8e42-113">Os exemplos incluem a letra grega rho (ρ) e os códigos da entidade HTML, como por exemplo, \&eacute; para um *e* que tenha um acento agudo (é).</span><span class="sxs-lookup"><span data-stu-id="f8e42-113">Examples include the Greek letter rho (ρ) and HTML entity codes such as \&eacute; for an *e* that has an acute accent (é).</span></span>

> [!NOTE]
> <span data-ttu-id="f8e42-114">Saiba que a estrutura não é controlada se o conteúdo XML colocado para o documento gerado usando o elemento do formato **RAW XML** estiver correto.</span><span class="sxs-lookup"><span data-stu-id="f8e42-114">Be aware that the framework doesn't control whether the XML content that is placed to the generated document by using the **RAW XML** format element is correct.</span></span>
