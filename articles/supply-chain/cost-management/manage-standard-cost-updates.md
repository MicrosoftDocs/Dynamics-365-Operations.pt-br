[!INCLUDE [banner](../includes/banner.md)]

<span data-ttu-id="f49f7-104">As atualizações de dados de custo padrão podem ser gerenciadas usando duas abordagens diferentes: a abordagem de uma versão ou a abordagem de duas versões.</span><span class="sxs-lookup"><span data-stu-id="f49f7-104">Updates to standard cost data can be managed by using two different approaches - the one-version approach or the two-version approach.</span></span> 

<span data-ttu-id="f49f7-105">A abordagem de uma versão usa uma única versão de avaliação de custo que contém todos os registros de custo.</span><span class="sxs-lookup"><span data-stu-id="f49f7-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="f49f7-106">Esses registros incluem os custos originais e todas as atualizações de custo.</span><span class="sxs-lookup"><span data-stu-id="f49f7-106">These records include the original costs and all cost updates.</span></span>

<span data-ttu-id="f49f7-107">A abordagem de duas versões usa uma versão que contém os registros de custos originais e uma segunda versão que contém os registros de todas as atualizações de custo.</span><span class="sxs-lookup"><span data-stu-id="f49f7-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="f49f7-108">Uma vantagem principal da abordagem de duas versões é a delineação clara e o rastreamento de atualizações de custo em uma versão de custo separada sem afetar a versão de custo original.</span><span class="sxs-lookup"><span data-stu-id="f49f7-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="f49f7-109">A abordagem de duas versões pode ser usada para identificar várias atualizações incrementais, onde cada atualização incremental possui uma versão de custo separada que contém os registros incrementais de custo.</span><span class="sxs-lookup"><span data-stu-id="f49f7-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span>
