## <a name="company-concept-in-common-data-service"></a><span data-ttu-id="e2e81-103">Conceito de empresa no Common Data Service</span><span class="sxs-lookup"><span data-stu-id="e2e81-103">Company concept in Common Data Service</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="e2e81-104">No Finance and Operations, o conceito de *empresa* é um conceito legal e um conceito comercial.</span><span class="sxs-lookup"><span data-stu-id="e2e81-104">In Finance and Operations, the concept of a *company* is both a legal construct and a business construct.</span></span> <span data-ttu-id="e2e81-105">Ele também é um limite de segurança e de visibilidade de dados.</span><span class="sxs-lookup"><span data-stu-id="e2e81-105">It's also a security and visibility boundary for data.</span></span> <span data-ttu-id="e2e81-106">Os usuários trabalham sempre no contexto de uma única empresa e a maioria dos dados são distribuídos por empresa.</span><span class="sxs-lookup"><span data-stu-id="e2e81-106">Users always work in the context of a single company, and most of the data is striped by company.</span></span>

<span data-ttu-id="e2e81-107">O Common Data Service não tem um conceito equivalente.</span><span class="sxs-lookup"><span data-stu-id="e2e81-107">Common Data Service doesn't have an equivalent concept.</span></span> <span data-ttu-id="e2e81-108">O conceito mais próximo é *unidade de negócios*, que é basicamente um limite de segurança e de visibilidade para dados de usuário.</span><span class="sxs-lookup"><span data-stu-id="e2e81-108">The closest concept is *business unit*, which is primarily a security and visibility boundary for user data.</span></span> <span data-ttu-id="e2e81-109">Esse conceito não tem as mesmas implicações legais ou comerciais que o conceito de empresa.</span><span class="sxs-lookup"><span data-stu-id="e2e81-109">This concept doesn't have the same legal or business implications as the company concept.</span></span>

<span data-ttu-id="e2e81-110">Como a unidade de negócios e a empresa não são conceitos equivalentes, não é possível forçar um mapeamento linear (1:1) entre elas com a finalidade de integração do Common Data Service.</span><span class="sxs-lookup"><span data-stu-id="e2e81-110">Because business unit and company aren't equivalent concepts, it isn't possible to force a one-to-one (1:1) mapping between them for the purpose of Common Data Service integration.</span></span> <span data-ttu-id="e2e81-111">Porém, como os usuários devem, por padrão, ser capazes de ver os mesmos registros no aplicativo e no Common Data Service, a Microsoft apresentou uma nova entidade no Common Data Service, chamada cdm\_Company.</span><span class="sxs-lookup"><span data-stu-id="e2e81-111">However, because users must, by default, be able to see the same records in the application and Common Data Service, Microsoft has introduced a new entity in Common Data Service that is named cdm\_Company.</span></span> <span data-ttu-id="e2e81-112">Essa entidade é equivalente à entidade Empresa no aplicativo.</span><span class="sxs-lookup"><span data-stu-id="e2e81-112">This entity is equivalent to the Company entity in the application.</span></span> <span data-ttu-id="e2e81-113">Para ajudar a garantir que a visibilidade dos registros seja imediatamente equivalente entre o aplicativo e o Common Data Service, recomendamos a seguinte configuração para dados no Common Data Service:</span><span class="sxs-lookup"><span data-stu-id="e2e81-113">To help guarantee that visibility of records is equivalent between the application and Common Data Service out of the box, we recommend the following setup for data in Common Data Service:</span></span>

+ <span data-ttu-id="e2e81-114">Para cada registro de empresa do Finance and Operations que é habilitado para gravação dupla, é criado um registro associado cdm\_Company.</span><span class="sxs-lookup"><span data-stu-id="e2e81-114">For each Finance and Operations Company record that is enabled for dual-write, an associated cdm\_Company record is created.</span></span>
+ <span data-ttu-id="e2e81-115">Quando um registro cdm\_Company é criado e habilitado para gravação dupla, uma unidade de negócios padrão é criada com o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="e2e81-115">When a cdm\_Company record is created and enabled for dual-write, a default business unit is created that has the same name.</span></span> <span data-ttu-id="e2e81-116">Embora uma equipe padrão seja criada automaticamente para essa unidade de negócios, a unidade de negócios não é usada.</span><span class="sxs-lookup"><span data-stu-id="e2e81-116">Although a default team is automatically created for that business unit, the business unit isn't used.</span></span>
+ <span data-ttu-id="e2e81-117">Uma equipe de proprietário separada é criada com o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="e2e81-117">A separate owner team is created that has the same name.</span></span> <span data-ttu-id="e2e81-118">Ela também é associada à unidade de negócios.</span><span class="sxs-lookup"><span data-stu-id="e2e81-118">It's also associated with the business unit.</span></span>
+ <span data-ttu-id="e2e81-119">Por padrão, o proprietário de qualquer registro criado e com gravação dupla no Common Data Service é definido para a equipe "Proprietário DW" vinculada à unidade de negócios associada.</span><span class="sxs-lookup"><span data-stu-id="e2e81-119">By default, the owner of any record that is created and dual-written to Common Data Service is set to the "DW Owner" team that is linked to the associated business unit.</span></span>
