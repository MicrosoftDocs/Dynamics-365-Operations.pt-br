# <a name="create-a-performance-review"></a>Criar uma avaliação de desempenho

[!include [task guide banner](../../includes/task-guide-banner.md)]

Este procedimento mostra como criar uma previsão dos resultados e descreve a finalidade de cada seção de revisão. Este procedimento foi criado usando a empresa de dados de demonstração USMF. Este procedimento é para um recurso que foi adicionado na versão 1611 do Dynamics 365 for Operations.

1. Clique em Autoatendimento para funcionários.
2. Clique em Nova revisão para criar nova revisão.
3. No campo Tipo de revisão, insira ou selecione um valor.
4. No campo Período de desempenho, insira ou selecione um valor.
5. No campo Data final, insira uma data.
6. Clique em OK.
    * Você também pode criar uma revisão a partir de um modelo. Essa é a melhor maneira de criar uma revisão, já que cada ação conterá as informações que você precisa para iniciar uma revisão.  
7. Clique em Exibir seções para abrir a caixa de diálogo suspensa.
8. Selecione Não no campo Exibir anexos.
9. Clique em Salvar.
    * Observe que a guia de anexos agora está oculta.  
10. Clique em Exibir seções para abrir a caixa de diálogo suspensa.
11. Selecione Sim no campo Exibir anexos.
12. Clique em Salvar.
13. Clique em Adicionar meta para revisão.
14. Clique em Cancelar.
15. Clique em Adicionar competência para abrir a caixa de diálogo suspensa.
16. No campo Título, digite um valor.
17. No campo descrição, insira "habilidades do aumento trabalhando com a equipe de suporte".
18. Clique em OK.
19. Clique em Recolher Tudo.
20. Clique em Expandir Tudo.
21. Clique em Adicionar comentário.
22. Clique em Lançar.
23. Clique na guia Medidas.
24. Clique em Adicionar medida para abrir a caixa de diálogo suspensa.
25. No campo Medida, insira ou selecione um valor.
26. No campo Valor alvo, insira um número.
27. Clique em OK.
28. Clique na guia Atividades.
29. Clique em Adicionar.
30. No campo Título, digite um valor.
31. No campo Descrição, digite um valor.
32. No campo Data de início, insira uma data.
33. No campo Data de conclusão, insira uma data.
34. Selecione Sim no campo de Desenvolvimento do plano.
35. No campo Palavras-chave, digite um valor.
36. Clique em Salvar.
37. Clique na guia Avaliações.
    * A FastTab de detalhes de classificação permite que os funcionários se avaliem e que o gerente avalie os funcionários. Se pesos forem usados, o valor do peso dos pontos será calculado automaticamente.    Para ver essa seção, habilite as configurações de parâmetro para exibição das classificações do funcionário.  
38. Clique na guia Desconexões.
    * Se a revisão usar um fluxo de trabalho, as aprovações aparecerão só depois da conclusão do fluxo de trabalho. Se nenhum fluxo de trabalho for usado, o trabalhador e o gerente estarão listados aqui. A caixa de seleção necessária é selecionada com base nas configurações do tipo de revisão.  
39. Clique na guia Geral.
    * O período de desempenho cria datas inicial e final padrão. Essas datas são editáveis.  
    * Os status controlam o acesso à revisão. O status não iniciado permite que todos editem a revisão. O status em andamento permite que o funcionário só exiba e edite a revisão. Apronte para revisão permitir que somente o gerenciador exiba e edite a revisão. Status final da revisão permite que o funcionário e gerente exibam a revisão e também editá-la configuração se o tipo de análise. Os status Concluído, Rejeitado e Cancelado deixam a revisão apenas leitura.  
40. No campo Visão geral, digite um valor.
41. Clique na guia Revisão.
    * Como revisão se move com o status, o funcionário e gerente podem adicionar comentários ou para cada meta competência.  
42. Clique na guia Desconexões.
    * O trabalhador e gerente podem se desconectar da revisão. Quando todos os prazos de conexão exigidos for concluído, o status será alterado para concluído e não poderá mais alteração ser feita.
